## 🎯 О проекте

Steam Farm Bot - это мощный Telegram бот для автоматического накручивания игрового времени в Steam. Работает 24/7, поддерживает множество аккаунтов и игр одновременно.

### Ключевые преимущества

- ⚡ **Высокая производительность** - до 100 одновременных сессий
- 🔒 **Безопасность** - поддержка Steam Guard и QR-авторизации
- 📊 **Детальная статистика** - отслеживание прогресса в реальном времени
- 🐳 **Docker Ready** - простое развертывание в контейнерах
- 🔄 **Автовосстановление** - автоматический перезапуск при сбоях

## ✨ Возможности

### Основной функционал

- 🎮 **Мульти-фарм** - до 30 игр одновременно на одном аккаунте
- 👻 **Режим невидимки** - фарм в режиме "Не в сети"
- 💬 **Кастомный статус** - установка собственного названия игры
- ⏱️ **Точное время** - отображение в формате ЧЧ:ММ:СС
- 📈 **Прогресс по играм** - начальные → текущие часы (+нафармлено)

### Управление аккаунтами

- 🚀 **Быстрая авторизация** - QR-код или логин:пароль
- 🔐 **Steam Guard** - полная поддержка двухфакторной аутентификации
- 🔄 **Автоперезапуск** - восстановление сессий при отключении
- 📚 **Библиотека игр** - автоматическая загрузка из Steam
- 🎯 **Цели фарма** - установка и отслеживание целей

### Дополнительно

- 💼 **Автотрейды** - автоматическое принятие трейдов-подарков
- 🔔 **Уведомления** - трейды, заявки в друзья, события
- 💳 **Подписки** - Premium тарифы с увеличенными лимитами
- 👥 **Реферальная система** - 10% от покупок приглашенных
- 📊 **Статистика** - топы пользователей, игр и аккаунтов

## 📦 Установка

### Требования

- **Node.js** 18.0.0 или выше
- **npm** или **yarn**
- **Docker** (опционально, но рекомендуется)
- **Telegram Bot Token** - получить у [@BotFather](https://t.me/BotFather)

### Docker установка (Рекомендуется)

Бот использует микросервисную архитектуру с 3 контейнерами:

```yaml
services:
  database:    # Автоматические бэкапы каждый час
  bot:         # Telegram бот (обработка команд)
  farm:        # Логика фарма Steam аккаунтов
```

#### 1. Клонирование репозитория

```bash
git clone https://github.com/yourusername/steam-farm-bot.git
cd steam-farm-bot
```

#### 2. Настройка переменных окружения

```bash
cp .env.example .env
```

Отредактируйте `.env`:

```env
# Обязательные параметры
BOT_TOKEN=your_telegram_bot_token
ADMIN_IDS=123456789,987654321
BOT_USERNAME=YourBotUsername

# Опциональные параметры
PROXY_URL=http://username:password@host:port
CRYPTO_API_TOKEN=your_crypto_bot_token
STEAM_WEB_API_KEY=your_steam_api_key
FORCE_PROXY=false
```

#### 3. Сборка и запуск

```bash
# Сборка образов
docker-compose build

# Запуск всех сервисов
docker-compose up -d

# Проверка статуса
docker-compose ps
```

#### 4. Управление контейнерами

```bash
# Просмотр логов
docker-compose logs -f bot     # Логи бота
docker-compose logs -f farm    # Логи фарм-сервиса
docker-compose logs -f         # Все логи

# Перезапуск
docker-compose restart

# Остановка
docker-compose down

# Пересборка без кеша
docker-compose build --no-cache
```

### Локальная установка

#### 1. Установка зависимостей

#### 2. Настройка .env

cp .env.example .env

## ⚙️ Конфигурация

### Переменные окружения

| Переменная | Описание | Обязательна | По умолчанию |
|-----------|----------|-------------|--------------|
| `BOT_TOKEN` | Telegram Bot Token | ✅ | - |
| `ADMIN_IDS` | ID администраторов (через запятую) | ✅ | - |
| `BOT_USERNAME` | Username бота | ✅ | - |
| `PROXY_URL` | URL прокси-сервера | ❌ | - |
| `FORCE_PROXY` | Принудительное использование прокси | ❌ | `false` |
| `CRYPTO_API_TOKEN` | Токен CryptoBot API | ❌ | - |
| `STEAM_WEB_API_KEY` | Steam Web API ключ | ❌ | - |
| `DB_PATH` | Путь к базе данных | ❌ | `./database.db` |

### Получение токенов

#### Telegram Bot Token

1. Откройте [@BotFather](https://t.me/BotFather)
2. Отправьте `/newbot`
3. Следуйте инструкциям
4. Скопируйте полученный токен

#### Telegram ID

1. Откройте [@userinfobot](https://t.me/userinfobot)
2. Отправьте `/start`
3. Скопируйте ваш ID

#### Steam Web API Key (опционально)

1. Откройте [Steam Web API Key](https://steamcommunity.com/dev/apikey)
2. Войдите в аккаунт Steam
3. Заполните форму и получите ключ

## 🎯 Использование

### Команды бота

| Команда | Описание |
|---------|----------|
| `/start` | Запуск бота и главное меню |
| `/menu` | Открыть главное меню |
| `/admin` | Админ-панель (только для администраторов) |

### Основной workflow

#### 1. Добавление аккаунта

**Способ 1: QR-код (рекомендуется)**
- Нажмите "Добавить аккаунт" → "QR-код"
- Отсканируйте QR-код в мобильном приложении Steam
- Готово! Аккаунт добавлен

**Способ 2: Логин и пароль**
- Нажмите "Добавить аккаунт" → "Логин и пароль"
- Введите логин
- Введите пароль
- При необходимости введите Steam Guard код

**Способ 3: Быстрый ввод**
- Отправьте сообщение в формате: `логин:пароль`
- Бот автоматически распознает и добавит аккаунт

#### 2. Выбор игр

- Откройте "Мои аккаунты" → выберите аккаунт
- Нажмите "Управление играми"
- Выберите игры из библиотеки или добавьте по App ID
- Можно выбрать топ-10 игр по времени

#### 3. Запуск фарма

- Откройте аккаунт
- Нажмите "Запустить фарм"
- Фарм начнется автоматически
- Отслеживайте прогресс в реальном времени

#### 4. Мониторинг

- Время отображается в формате `ЧЧ:ММ:СС`
- Для каждой игры показывается: `начальные → текущие (+нафармлено)`
- Статус аккаунта: 🟢 Фармит / ⚫ Остановлен
- Режим: 🌐 В сети / 👻 Невидимка

### Дополнительные функции

#### Кастомный статус

- Откройте аккаунт → "Настройки"
- Выберите "Кастомный статус"
- Введите текст (например: "Grand Theft Auto VI")
- Статус будет отображаться в Steam

#### Режим невидимки

- Откройте аккаунт → "Настройки"
- Переключите "Режим невидимки"
- Аккаунт будет фармить в режиме "Не в сети"

#### Автотрейды

- Откройте аккаунт → "Настройки"
- Включите "Автопринятие трейдов"
- Бот будет автоматически принимать трейды-подарки

#### Уведомления

- Откройте "Настройки" → "Уведомления"
- Выберите типы уведомлений:
  - Трейды
  - Заявки в друзья
  - Достижения целей
  - Ошибки фарма

### База данных

SQLite база данных с автоматическими бэкапами:

- **Бэкапы**: каждый час
- **Хранение**: 7 дней
- **Оптимизация**: каждые 30 минут
- **Индексы**: 4+ индекса для быстрых запросов

#### Восстановление из бэкапа

```bash
# Список бэкапов
docker run --rm -v steam-farm-bot_db-backups:/backups alpine ls -lh /backups

# Восстановление
docker run --rm \
  -v steam-farm-bot_db-backups:/backups \
  -v steam-farm-bot_db-data:/data \
  alpine cp /backups/database-YYYYMMDD-HHMMSS.db /data/database.db

# Перезапуск сервисов
docker-compose restart
```

## ❓ FAQ

### Общие вопросы

**Q: Безопасно ли использовать бота?**  
A: Да, бот использует официальные Steam API и не нарушает правила Steam. Однако используйте на свой риск.

**Q: Сколько аккаунтов можно добавить?**  
A: Зависит от подписки:
- Триал: 5 аккаунтов
- Базовый: 15 аккаунтов
- Полный: 50 аккаунтов

**Q: Сколько игр можно фармить одновременно?**  
A: До 30 игр на одном аккаунте.

**Q: Работает ли бот 24/7?**  
A: Да, при использовании Docker контейнеров бот работает круглосуточно.

### Технические вопросы

**Q: Как обновить бота?**  
A: 
```bash
git pull origin main
docker-compose build --no-cache
docker-compose up -d
```

**Q: Как посмотреть логи?**  
A:
```bash
docker-compose logs -f bot    # Логи бота
docker-compose logs -f farm   # Логи фарм-сервиса
```

**Q: Что делать при ошибке LogonSessionReplaced?**  
A: Это исправлено в версии 1.9.5.1. Обновите бота до последней версии.

**Q: Как увеличить лимиты?**  
A: Приобретите Premium подписку через бота: Меню → Подписка
