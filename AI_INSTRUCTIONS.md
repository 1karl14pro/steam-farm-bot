## 🔧 СПЕЦИФИЧЕСКИЕ ПРАВИЛА

### 1. Работа с сессиями Steam

#### Важно знать:
- **LogonSessionReplaced** - возникает при одновременном входе с одного аккаунта
- **Задержка между запусками** - минимум 5 секунд (sessionManager.js:79)
- **Автоперезапуск** - через 30 секунд после отключения
- **Максимум сессий** - 100 одновременных (farmManager.js:11)

#### При изменении логики сессий:
1. Проверить, не создается ли новая сессия при активном фарме
2. Убедиться, что задержка между запусками достаточная
3. Проверить обработку LogonSessionReplaced
4. Протестировать восстановление после перезагрузки

### 2. Работа с форматированием

#### Отображение времени:
- Формат: `ЧЧ:ММ:СС` (например: `11:23:45`)
- Файл: `src/services/formatter.js`
- Функции: `formatAccountInfo`, `formatUserProfileFull`

#### Отображение статистики игр:
- Формат: `начальные → текущие (+нафармлено)`
- Пример: `📊 10.5ч → 11.2ч (+0.7ч)`
- Функция: `getGamesWithHours` в database.js

### 3. Работа с обновлениями

#### При обновлении версии:
1. Обновить `package.json` (version)
2. Обновить `README.md` (раздел "Последние обновления")
3. Добавить запись в `CHANGELOG.md`
4. Создать коммит: `chore: bump version to X.X.X`
5. Отправить на GitHub
6. Обновить VDS

#### Формат версий:
- **MAJOR** - несовместимые изменения API
- **MINOR** - новые функции (обратно совместимые)
- **PATCH** - исправления ошибок
- **HOTFIX** - критические исправления

Пример: `1.9.5.1`
- 1 - мажорная версия
- 9 - минорная версия
- 5 - патч
- 1 - хотфикс
