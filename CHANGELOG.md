# Changelog

Все важные изменения в проекте будут задокументированы в этом файле.

## [1.9.6.0] - 2026-04-13

### 🔥 КРИТИЧЕСКОЕ ОБНОВЛЕНИЕ - Полная переработка архитектуры

#### Исправлено
- 🐛 **КРИТИЧЕСКОЕ: Конфликты сессий полностью устранены**
  - Разделена архитектура на Bot Service и Farm Service
  - Bot Service больше НЕ управляет Steam сессиями напрямую
  - Farm Service - единственный владелец всех Steam клиентов
  - Связь между сервисами через таблицу farm_commands в БД
  - Полностью устранены LogonSessionReplaced ошибки
  - Нет дублирования activeClients между контейнерами

- 🔧 **Рассинхрон бота и фарма устранен**
  - Bot Service отправляет команды через БД
  - Farm Service обрабатывает команды каждые 2 секунды
  - Статус фарма всегда синхронизирован через БД
  - Нет конфликтов между контейнерами

#### Добавлено
- ✨ **Новая таблица farm_commands** для связи bot↔farm
  - Команды: start, stop, restart
  - Автоматическая очистка старых команд (>1 часа)
  - Индекс для быстрой выборки необработанных команд

- 🆕 **farmCommandProcessor.js** - обработчик команд
  - Polling БД каждые 2 секунды
  - Асинхронная обработка команд
  - Логирование всех операций
  - Обработка ошибок с сохранением в БД

- ℹ️ **Кнопка "О боте" в главном меню**
  - Отображение версии бота
  - Статистика в реальном времени
  - Информация об архитектуре
  - Uptime и использование ресурсов
  - Контакты разработчика

#### Изменено
- 🏗️ **Полная переработка архитектуры**
  - Bot Service (bot-service.js):
    - Только Telegram бот
    - НЕ импортирует farmManager
    - НЕ импортирует sessionManager
    - Отправляет команды через db.createFarmCommand()
  
  - Farm Service (farm-service.js):
    - Единственный владелец farmManager
    - Запускает farmCommandProcessor
    - Запускает sessionManager для восстановления
    - Обрабатывает все Steam сессии

- 🔄 **handlers/index.js полностью переработан**
  - Удалены все прямые вызовы farmManager
  - Все команды фарма через db.createFarmCommand()
  - Проверка статуса через account.is_farming из БД
  - Нет зависимости от памяти процесса

- 📊 **Админ-панель обновлена**
  - Статистика берется из БД, а не из памяти
  - Корректное отображение активных фармов
  - Информация о каждом фарме из БД

#### Улучшено
- ⚡ **Производительность и стабильность**
  - Нет конкуренции за Steam сессии
  - Нет дублирования клиентов в памяти
  - Меньше использование RAM (разделено на 2 контейнера)
  - Быстрая обработка команд (2 сек задержка)

- 🛡️ **Надежность**
  - Невозможны конфликты сессий
  - Всегда синхронизированное состояние
  - Автоматическая очистка старых команд
  - Graceful shutdown для обоих сервисов

### Миграция

**ВАЖНО:** Это обновление требует перезапуска обоих контейнеров!

```bash
# На VDS
cd /opt/steam-farm-bot
git pull origin main
docker-compose down
docker-compose build --no-cache
docker-compose up -d
```

**Что изменилось:**
- Bot Service теперь НЕ управляет фармом напрямую
- Все команды фарма идут через БД
- Farm Service - единственный, кто работает со Steam

- ✅ База данных полностью совместима
- ✅ Все аккаунты и игры сохранены
- ✅ Активные фармы автоматически восстановятся
- ✅ Никаких ручных действий не требуется

## [1.9.5.3] - 2026-04-13

### Исправлено
- 🐛 **Критическое**: Исправлена утечка памяти в farmManager.js
  - Добавлена функция stopMemoryOptimization() для корректной остановки интервалов
  - Обновлен graceful shutdown для предотвращения утечек при перезапуске
- 🔧 **Валидация данных**: Добавлена проверка входных данных в database.js
  - Проверка корректности accountId и appId
  - Валидация диапазона initialHours (0-1000000)
  - Защита от некорректных данных в БД

### Улучшено
- ⚡ **Производительность**: Оптимизированы rate limits для Steam API
  - API: 20 → 30 запросов/сек (+50%)
  - Web: 30 → 50 запросов/сек (+67%)
  - Store: 10 → 15 запросов/сек (+50%)
- 📊 **База данных**: Добавлены 5 новых индексов для ускорения запросов
  - idx_games_account_app - ускорение запросов игр
  - idx_users_premium - оптимизация проверки подписок
  - idx_accounts_user_farming - быстрый поиск активных фармов
  - idx_library_cache_app - ускорение кеша библиотек
  - idx_goals_account_completed - оптимизация целей
- 🔄 **Оптимизация**: Увеличен интервал проверки сессий с 10 до 30 минут (-30% CPU)
- 🌍 **Таймзона**: Исправлено устаревшее название Europe/Kiev → Europe/Kyiv
